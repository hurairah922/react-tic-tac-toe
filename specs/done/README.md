# Done Features

This directory keeps a short record of completed phases and feature work.

It is shared documentation for the project:

- You can update it manually.
- The agent can update it when a phase is finished.
- It should reflect what was actually completed, not just what was planned.

## Suggested Workflow

1. Keep the current feature spec in `specs/features/active.md` while work is in progress.
2. When the phase is complete, set the phase status to `Done` in `specs/phases.md`.
3. Add or update a matching file in this directory with the delivered feature details.
4. Include follow-up notes if anything was intentionally left for a later phase.

## Suggested File Naming

- `phase-00-specs-and-baseline.md`
- `phase-01-learn-how-to-play-modal.md`
- `phase-02-dynamic-board-rules-foundation.md`

## Suggested Sections

- Summary
- What shipped
- Acceptance criteria covered
- Notes
- Follow-ups

Use as much detail as is useful, but keep it focused on what was actually delivered.
